# 화면 상세 기획서: 분양 전용 서비스

## 1. 문서 목적

- PRD를 기준으로 실제 화면 구조와 사용자 행동 흐름을 구체화한다.
- MVP에서 반드시 필요한 화면과 기능 범위를 명확히 한다.
- 이후 와이어프레임, UI 디자인, 개발 명세의 기준 문서로 사용한다.

## 2. 서비스 구조 요약

### 사용자의 핵심 목표
- 지금 청약 가능한 분양 단지를 찾는다.
- 내 조건으로 의미 있는 단지만 본다.
- 분양 일정과 공급 유형을 빠르게 이해한다.
- 관심 단지를 저장하고 청약 일정을 관리한다.

### 핵심 내비게이션
- 홈
- 분양 찾기
- 일정
- 관심 단지
- 마이

### 추천 정보 구조 원칙
- 매매/전월세 관련 메뉴는 두지 않는다.
- 사용자가 지금 행동할 수 있는 정보부터 보여준다.
- 단지 소개보다 청약 실행 정보가 먼저 보이게 한다.

## 3. 핵심 사용자 플로우

### 플로우 A: 분양 탐색 후 상세 확인
1. 사용자가 홈 또는 분양 찾기에 진입한다.
2. 분양 단계 필터를 선택한다.
3. 지역, 예산, 공급 유형 필터를 추가한다.
4. 리스트에서 단지를 선택한다.
5. 상세에서 일정, 분양가, 공급 유형, 주의사항을 확인한다.

### 플로우 B: 내 조건으로 청약 가능 단지 찾기
1. 사용자가 홈에서 `내 조건 설정` 또는 `청약 자격 진단`으로 이동한다.
2. 무주택 여부, 거주 지역, 청약통장 여부, 특별공급 대상 여부를 입력한다.
3. 서비스가 조건과 맞는 단지를 추천한다.
4. 상세 화면에서 공급 유형별 가능 여부를 확인한다.

### 플로우 C: 관심 단지 저장 후 일정 관리
1. 사용자가 리스트 또는 상세에서 관심 단지를 저장한다.
2. 서비스는 관심 단지 일정만 모아 보여준다.
3. 사용자는 일정 알림을 활성화한다.
4. 일정 화면에서 다가오는 접수일, 발표일, 계약일을 확인한다.

## 4. 화면 목록

- 온보딩
- 홈
- 분양 찾기(리스트 뷰)
- 분양 찾기(지도 뷰)
- 필터 바텀시트 또는 필터 패널
- 단지 상세
- 청약 자격 진단
- 청약 가점 계산기
- 관심 단지
- 비교함
- 일정 캘린더
- 알림 설정
- 마이페이지

## 5. 화면별 상세 기획

## 5.0 온보딩

### 화면 목적
- 신규 사용자가 서비스의 핵심 가치를 이해하고, 맞춤 추천에 필요한 기본 조건을 자연스럽게 입력하게 한다.
- 첫 진입부터 개인화된 경험을 제공해 초기 이탈을 줄인다.

### 단계 구성
- 1단계: 서비스 소개 (2~3 스와이프 카드)
  - "분양과 청약에 필요한 정보만 모았어요"
  - "내 조건에 맞는 단지를 추천받으세요"
  - "일정 알림으로 청약을 놓치지 마세요"
- 2단계: 관심 지역 선택 (다중 선택)
- 3단계: 예산 범위 입력 (슬라이더)
- 4단계: 기본 조건 입력 (무주택 여부, 세대주 여부, 청약통장 유무)
- 5단계: 특별공급 대상 선택 (신혼부부, 생애최초, 다자녀, 해당 없음)
- 완료: 맞춤 추천 결과 홈으로 이동

### 주요 액션
- 다음 단계 이동
- 건너뛰기 (모든 단계에서 가능)
- 완료

### UX 메모
- 전체 소요 시간 1분 이내로 설계한다.
- 프로그레스 바로 현재 진행 상황을 보여준다.
- 건너뛴 항목은 홈의 조건 카드에서 추후 입력을 유도한다.
- 비로그인 상태에서도 입력 가능하고, localStorage에 저장한 뒤 로그인 시 서버 동기화한다.

## 5.1 홈

### 화면 목적
- 사용자가 현재 시점에서 가장 중요한 분양 정보를 빠르게 파악하게 한다.
- 탐색 시작점과 개인화 진입점을 동시에 제공한다.

### 주요 섹션
- 상단 검색/지역 선택 영역
- 내 조건 요약 카드
- 지금 청약 가능한 단지
- 곧 모집공고가 나오는 단지
- 특별공급 진행 단지
- 내 조건에 맞는 추천 단지
- 일정 임박 단지

### 필수 UI 요소
- 검색창: 단지명, 지역명 검색
- 빠른 필터 칩: 분양 예정, 모집공고, 특별공급, 1순위, 발표 예정
- 내 조건 설정 버튼
- 각 섹션 더보기 버튼

### 주요 액션
- 검색어 입력
- 단계 칩 선택
- 내 조건 입력/수정
- 단지 카드 클릭
- 관심 단지 저장

### 노출 우선순위
1. 지금 접수 가능한 단지
2. 일정이 임박한 단지
3. 개인 조건에 맞는 단지
4. 곧 공고가 나오는 단지

### 빈 상태
- 추천 조건이 없으면 `조건을 입력하면 맞춤 단지를 추천해드려요` 안내 노출

## 5.2 분양 찾기(리스트)

### 화면 목적
- 사용자가 분양 단계와 청약 맥락에 맞는 필터로 단지를 탐색하게 한다.

### 상단 영역
- 검색창
- 현재 적용 중인 분양 단계 필터
- 정렬 옵션
- 필터 버튼

### 기본 정렬
- 접수 가능 순
- 일정 임박 순
- 최신 공고 순

### 리스트 카드 정보
- 단지명
- 행정구역
- 분양 단계 배지
- 공급 유형
- 대표 분양가
- 전용면적 범위
- 접수 일정 또는 발표일
- 관심 저장 버튼

### 상단 고정 필터
- 분양 단계
- 지역
- 공급 유형
- 분양가 범위

### 추가 필터
- 전용면적
- 일반공급/특별공급
- 특별공급 유형
- 규제지역 여부
- 입주 예정 시기
- 건설사

### 주요 액션
- 필터 적용/해제
- 정렬 변경
- 카드 클릭
- 비교함 추가

### UX 메모
- 리스트 상단에 결과 수를 표시한다.
- 필터가 적용되면 칩 형태로 노출하고 개별 삭제 가능해야 한다.
- 모바일에서는 바텀시트 필터, 웹에서는 좌측 패널 필터 구조를 우선 고려한다.

## 5.2.1 분양 찾기(지도 뷰)

### 화면 목적
- 사용자가 관심 지역의 분양 단지를 지도 위에서 직관적으로 탐색하게 한다.

### 상단 영역
- 검색창
- 뷰 전환 토글 (리스트 ↔ 지도)
- 필터 버튼

### 지도 영역
- 지도 전체 화면 표시
- 분양 단지 마커 (단계별 색상 구분)
- 마커 클러스터링 (축소 시 인근 단지 그룹화)
- 현재 지도 영역 기준 결과 수 표시

### 마커 클릭 시 간략 카드
- 단지명
- 분양 단계 배지
- 대표 분양가
- 접수 일정 또는 D-day
- 관심 저장 버튼
- 상세 이동 버튼

### 하단 슬라이드 패널
- 현재 지도 영역의 단지 목록 (카드 형태)
- 위로 스와이프하면 리스트로 확장
- 아래로 스와이프하면 지도 전체 화면

### 주요 액션
- 지도 이동/줌 시 자동으로 단지 목록 갱신
- 마커 클릭 → 간략 카드 → 상세 이동
- 뷰 전환
- 필터 적용 (리스트 뷰와 동일한 필터)

### UX 메모
- 지도 뷰에서도 분양 단계 칩 필터를 상단에 유지한다.
- 마커 색상 범례를 지도 하단에 배치해 학습을 돕는다.
- 지도 이동 후 「이 지역 재검색」 버튼으로 수동 갱신도 지원한다.

## 5.3 필터 패널

### 화면 목적
- 일반 부동산 검색이 아닌 분양 맥락의 필터 경험을 제공한다.

### 필터 그룹
- 분양 단계
- 지역
- 가격
- 면적
- 공급 유형
- 공급 대상
- 일정
- 기타 조건

### 상세 항목
- 분양 단계: 분양 예정, 공고 공개, 특별공급, 1순위, 2순위, 발표, 계약, 입주 예정
- 공급 유형: 민간분양, 공공분양
- 공급 대상: 일반공급, 신혼부부, 생애최초, 다자녀, 노부모부양

### 버튼
- 초기화
- 결과 보기

### UX 메모
- 분양 단계는 단일 선택이 아니라 다중 선택을 허용한다.
- 사용자의 조건과 충돌하는 필터는 경고 문구를 보여줄 수 있다.

## 5.4 단지 상세

### 화면 목적
- 청약 의사결정에 필요한 정보를 한 번에 제공한다.

### 섹션 구성
- 단지 기본 정보
- 규제 정보 요약 배지
- 현재 분양 단계 요약
- 일정 타임라인
- 공급 정보
- 분양가 정보
- 경쟁률 정보
- 청약 자격 간이 결과
- 자금 계획 요약
- 입지 요약
- 유의사항
- 원문 링크

### 상단 핵심 카드
- 단지명
- 위치
- 분양 단계
- 대표 일정 D-day
- 관심 저장
- 비교 추가

### 일정 타임라인 구성
- 모집공고일
- 특별공급 접수일
- 1순위 접수일
- 2순위 접수일
- 당첨자 발표일
- 계약일
- 입주 예정일

### 공급 정보 구성
- 공급 유형
- 총 세대수
- 일반공급 세대수
- 특별공급 세대수
- 타입별 전용면적

### 분양가 정보 구성
- 타입별 분양가
- 계약금
- 중도금
- 잔금
- 옵션 비용 유무

### 경쟁률 정보 블록
- 과거 유사 단지 평균 경쟁률 (동일 지역, 유사 면적/분양가)
- 해당 지역 최근 1년 평균 경쟁률
- 타입별 예상 경쟁률 범위
- 과거 유사 단지 당첨 가점 범위 (최저~최고)
- 내 가점 기준 당첨 가능성 안내 (가점 입력 시)
- 가점제/추첨제 비율 표시
- (접수 중인 경우) 실시간 경쟁률 - 후속 확장

### 규제 정보 배지 블록
- 분양가 상한제 적용 여부
- 전매제한 기간 (예: 소유권 이전 등기 시까지, 3년 등)
- 실거주 의무 기간
- 재당첨 제한 기간
- 투기과열지구/조정대상지역 여부
- 중도금 대출 가능 여부
- 각 항목 클릭 시 쉬운 해설 툴팁 제공

### 청약 자격 블록
- 내 조건 기준 가능 공급 유형
- 추가 확인 필요 조건
- 주의사항
- 자세히 진단하기 버튼
- 내 가점으로 이 단지 도전해볼 만한지 안내

### 자금 계획 요약 블록
- 타입 선택 드롭다운
- 분양가
- 계약금 / 중도금 / 잔금 금액 및 예상 시기
- 자금 시뮬레이션 열기 버튼 (후속 확장)

### 하단 액션
- 관심 단지 저장
- 알림 설정
- 공고문 보기

### UX 메모
- 가장 먼저 보여줄 정보는 `단지 소개`가 아니라 `지금 단계`, `다음 일정`, `지원 가능한 공급`이다.
- 긴 공고문은 핵심 요약 후 원문 링크로 연결한다.

## 5.5 청약 자격 진단

### 화면 목적
- 사용자가 자신의 조건으로 어떤 공급 유형을 노려볼 수 있는지 파악하게 한다.

### 입력 항목
- 거주 지역
- 무주택 여부
- 세대주 여부
- 청약통장 보유 여부
- 청약통장 가입 기간
- 특별공급 대상 여부
- 혼인 여부
- 자녀 수

### 결과 화면
- 지원 가능 유형
- 제한 가능 사유
- 추가 확인 필요 항목
- 추천 단지 보기

### UX 메모
- 질문 수를 최소화해 1분 내 완료 가능하게 설계한다.
- 결과는 확정 판단이 아니라 참고 정보임을 명확히 표기한다.
- 단지 상세 내부에서 바로 진단을 열 수 있어야 한다.

## 5.5.1 청약 가점 계산기

### 화면 목적
- 사용자가 자신의 청약 가점을 빠르게 계산하고, 해당 가점으로 당첨 가능성을 판단하게 한다.

### 입력 항목
- 무주택 기간 (만 30세 이후 또는 혼인 이후 무주택 기간)
- 부양가족 수 (배우자, 직계존속, 직계비속)
- 청약통장 가입 기간

### 결과 화면
- 총 가점 (84점 만점)
  - 무주택 기간 점수 (0~32점)
  - 부양가족 수 점수 (0~35점)
  - 청약통장 가입 기간 점수 (0~17점)
- 내 가점 위치 시각화 (전체 분포에서 내 위치)
- 해당 가점으로 당첨 가능성이 있는 관심 단지 목록
- 가점이 낮은 경우 추첨제 비율이 높은 추천 단지
- 과거 유사 지역/면적 당첨 가점 범위 참고

### 주요 액션
- 가점 계산 실행
- 결과 저장 (프로필에 연동)
- 추천 단지 보러가기
- 다시 계산하기

### UX 메모
- 각 항목별 점수 기준표를 접을 수 있는 안내로 제공한다.
- 입력 시 실시간으로 점수가 갱신되는 인터랙션을 제공한다.
- 가점제 vs 추첨제 비율 안내를 포함한다 (면적 85㎡ 이하: 가점제 40~100%, 추첨제 0~60% 등).
- 단지 상세에서 바로 진입 가능해야 한다.

## 5.6 관심 단지

### 화면 목적
- 사용자가 추려둔 단지를 다시 검토하고 일정 중심으로 관리하게 한다.

### 표시 정보
- 단지명
- 분양 단계
- 다음 주요 일정
- 대표 분양가
- 알림 설정 상태

### 주요 액션
- 저장 해제
- 비교함 추가
- 알림 켜기/끄기
- 상세 이동

### 정렬
- 일정 임박 순
- 저장 최신 순

## 5.7 비교함

### 화면 목적
- 여러 단지를 빠르게 비교해 선택을 돕는다.

### 비교 항목
- 위치
- 분양 단계
- 공급 유형
- 세대수
- 분양가
- 전용면적
- 접수 일정
- 입주 예정일

### UX 메모
- 최대 5개까지 비교 허용
- 차이가 큰 항목은 강조 표시

## 5.8 일정 캘린더

### 화면 목적
- 사용자가 관심 단지의 일정을 한 화면에서 관리하게 한다.

### 보기 방식
- 월간 캘린더 보기
- 리스트 보기

### 이벤트 종류
- 모집공고
- 특별공급
- 1순위
- 2순위
- 발표일
- 계약일

### 주요 액션
- 날짜별 일정 확인
- 알림 설정
- 단지 상세 이동

### UX 메모
- 오늘 기준으로 지난 일정과 예정 일정을 구분 표시한다.
- 일정 색상은 단계별로 고정해 학습 가능하게 한다.

## 5.9 알림 설정

### 화면 목적
- 사용자가 중요한 분양 일정을 놓치지 않게 한다.

### 설정 항목
- 모집공고 알림
- 접수 시작 1일 전 알림
- 접수 당일 알림
- 발표일 알림
- 계약일 알림

### 알림 대상
- 관심 단지 전체
- 특정 단지만 선택

## 5.10 마이페이지

### 화면 목적
- 사용자 조건과 알림, 관심 단지 설정을 관리하게 한다.

### 주요 섹션
- 내 조건 요약
- 청약 자격 입력 정보 수정
- 알림 설정
- 관심 지역 관리
- 최근 본 단지

## 6. 핵심 컴포넌트 정의

### 단지 카드
- 단지명
- 지역
- 단계 배지
- 분양가
- 일정
- 저장 버튼

### 단계 배지
- 색상과 텍스트로 현재 분양 상태 표시
- 예: `특별공급 진행`, `1순위 접수 예정`, `계약 진행 중`

### 일정 타임라인
- 현재 단계 강조
- 완료 단계와 예정 단계를 시각적으로 분리

### 조건 요약 카드
- 거주 지역
- 무주택 여부
- 특별공급 여부
- 수정 버튼

### 규제 정보 배지 바
- 분양가 상한제 / 전매제한 / 실거주 의무 / 투기과열지구 등 아이콘+텍스트
- 클릭 시 쉬운 해설 툴팁
- 해당 없는 항목은 회색 처리

### 경쟁률 요약 카드
- 과거 유사 단지 평균 경쟁률
- 당첨 가점 범위
- 내 가점 대비 위치 표시 (바 또는 게이지)

### 가점 점수 카드
- 총 가점 / 84점
- 항목별 세부 점수 바

### 지도 마커
- 분양 단계별 색상 구분
- 클러스터 마커 (숫자 표시)
- 클릭 시 간략 카드 팝업

## 7. 상태 설계

### 로딩 상태
- 리스트 스켈레톤
- 상세 상단 카드 스켈레톤

### 빈 상태
- 검색 결과 없음
- 관심 단지 없음
- 추천 조건 없음

### 에러 상태
- 일정 데이터 미갱신
- 공고문 링크 오류
- 네트워크 오류

### 안내 문구 예시
- `조건에 맞는 단지가 없어요. 지역이나 분양 단계를 다시 선택해보세요.`
- `최신 공고 기준으로 정보를 갱신 중입니다.`

- 외부 캘린더 연동

## 9. 개발 전달용 메모

### 화면 진입 우선순위
1. 홈
2. 분양 찾기
3. 단지 상세
4. 청약 자격 진단
5. 관심 단지
6. 일정

### MVP에서 중요한 상호작용
- 단계 필터 선택 후 리스트 갱신
- 리스트/지도 뷰 전환
- 지도 이동 시 단지 목록 연동
- 상세에서 관심 저장
- 상세에서 알림 설정
- 가점 계산 후 추천 단지 연동
- 진단 완료 후 추천 단지 이동
- 규제 배지 클릭 시 해설 툴팁

### 데이터 의존도가 높은 영역
- 분양 단계 자동 계산
- 일정 정확도
- 공급 유형 및 특별공급 정보
- 사용자 조건 기반 추천
- 과거 경쟁률 및 당첨 가점 데이터
- 규제 조건 (분양가 상한제, 전매제한 등)
- 위치 좌표 데이터 (지도 마커)
