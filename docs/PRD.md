# PRD: 분양 전용 부동산 서비스

## 1. 제품 개요

### 제품명(가칭)
- Bunyang Flow

### 한 줄 정의
- 매매/전월세를 제외하고, 분양과 청약에 필요한 정보만 단계별로 보여주는 서비스

### 제품 목표
- 사용자가 분양 단지를 빠르게 탐색하고, 현재 분양 단계와 청약 가능 여부를 이해하고, 실제 신청 일정까지 놓치지 않도록 돕는다.

## 2. 문제 정의

### 현재 사용자 불편
- 기존 부동산 서비스는 매매, 전세, 월세, 분양 정보가 뒤섞여 있어 분양만 따로 보기 어렵다.
- 분양 단지의 현재 상태가 불명확해 사용자가 지금 무엇을 할 수 있는지 알기 어렵다.
- 모집공고, 특별공급, 1순위, 당첨 발표, 계약 일정이 분산되어 있어 관리가 어렵다.
- 청약 자격과 공급 유형이 복잡해 내가 지원 가능한 단지인지 빠르게 판단하기 어렵다.
- 공고문은 길고 전문 용어가 많아 핵심 내용 파악에 시간이 걸린다.
- 분양 단계의 필터가 세밀하지 않아, 분양중에 모든 단계가 다 노출되어 파악이 어렵다.
- 청약 가점을 직접 계산하기 번거롭고, 내 가점으로 당첨 가능성이 있는지 판단하기 어렵다.
- 과거 경쟁률, 가점 커트라인 데이터가 분산되어 있어 비교 판단 근거가 부족하다.
- 분양가 상한제, 전매제한, 실거주 의무 등 규제 조건이 단지별로 다르지만 한눈에 파악하기 어렵다.
- 계약금, 중도금, 잔금 일정과 중도금 대출 가능 여부를 종합적으로 확인하기 어렵다.

### 해결하려는 핵심 문제
- 분양 관련 정보만 분리해서 보여준다.
- 단지를 분양 단계 중심으로 탐색할 수 있게 한다.
- 사용자의 조건에 따라 실제로 볼 만한 단지를 우선 노출한다.
- 일정과 알림을 통해 분양 실행까지 이어지게 만든다.
- 청약 가점 계산과 경쟁률 비교를 통해 현실적인 당첨 가능성을 판단하게 한다.
- 규제 조건과 자금 계획을 한곳에서 파악하게 해 의사결정 시간을 줄인다.

### 경쟁사 분석

#### 네이버 부동산
- 강점: 방대한 데이터, 높은 트래픽, 지도 기반 탐색, 브랜드 신뢰도
- 약점:
  - 분양 정보가 매매/전월세와 혼재되어 분양 전용 탐색 경험이 약함
  - 분양 단계 필터가 「분양중/분양예정/분양완료」 3단계로 지나치게 단순함
  - 특별공급/1순위/2순위 등 세부 단계 구분이 불가능
  - 청약 자격 진단 기능 부재
  - 경쟁률, 가점 커트라인 등 당첨 가능성 판단 정보 미제공
  - 중도금 대출, 전매제한 등 실질 의사결정 정보가 공고문에 묻혀 있음

#### 호갱노노
- 강점: 시세 데이터 기반 분석, 커뮤니티 활성화, 직관적 UI
- 약점:
  - 매매/시세 중심 서비스로 분양 탐색 경험이 부수적
  - 분양 필터 옵션이 제한적 (지역, 기본 상태 정도)
  - 분양 단계별 세부 필터 부재
  - 청약 가점 계산기, 자격 진단 등 청약 실행 보조 기능 미비
  - 일정 관리/알림 기능 약함

#### 청약홈 (한국부동산원)
- 강점: 공식 데이터 원천, 실제 청약 신청 플랫폼, 경쟁률 공식 데이터
- 약점:
  - UX가 정보 전달에 최적화되어 있지 않음
  - 탐색/비교/필터 경험이 매우 제한적
  - 모바일 사용성 낮음
  - 개인화, 추천, 알림 기능 부재

## 3. 타깃 사용자

### 1차 타깃
- 한국 시장의 실수요자
- 생애 최초 주택 구입을 고려하는 사용자
- 신혼부부, 무주택 세대주, 청약 초보 사용자
- 수도권 및 주요 광역시 중심으로 신규 분양을 찾는 사용자

### 2차 타깃
- 특별공급 대상 여부를 확인하고 싶은 사용자
- 청약 일정을 놓치지 않고 관리하고 싶은 사용자
- 여러 분양 단지를 비교 검토하려는 사용자

### 제외 대상
- 전월세 매물 탐색 중심 사용자
- 일반 매매 거래 중심 사용자
- 기업용 분양 광고/마케팅 관리자

## 4. 제품 포지셔닝

### 포지셔닝 문장
- 이 서비스는 일반 부동산 플랫폼이 아니라, 분양과 청약 실행을 위한 전용 서비스다.

### 차별점
- 매매/전월세 정보 제거
- 분양 단계 기반 탐색 경험 (8단계 세부 필터)
- 청약 문맥에 맞는 필터 구조
- 일정 관리와 알림 중심 UX
- 분양 초보자를 위한 쉬운 해설
- 청약 가점 계산기와 경쟁률 기반 당첨 가능성 판단
- 규제 조건(분양가 상한제, 전매제한, 실거주 의무) 요약 제공
- 자금 계획 시뮬레이션 (계약금~잔금 납입 일정)
- 지도 기반 분양 단지 탐색

## 5. 핵심 가치 제안

### 사용자 가치
- 지금 청약 가능한 단지를 빠르게 찾을 수 있다.
- 내 조건으로 볼 의미가 있는 단지를 걸러볼 수 있다.
- 분양 단계와 일정 흐름을 한 번에 이해할 수 있다.
- 분양 공고의 핵심 정보를 쉽게 파악할 수 있다.
- 일정 누락 없이 실제 청약 행동으로 이어질 수 있다.

## 6. 핵심 사용자 시나리오

### 시나리오 1: 청약 가능한 단지 탐색
1. 사용자는 지역과 예산을 선택한다.
2. 서비스는 분양 단계 필터를 기준으로 청약 가능한 단지를 보여준다.
3. 사용자는 단지 상세에서 공급 유형, 분양가, 접수 일정, 입주 예정일을 확인한다.

### 시나리오 2: 내 조건에 맞는 단지 확인
1. 사용자는 무주택 여부, 거주 지역, 특별공급 대상 여부 등 기본 조건을 입력한다.
2. 서비스는 조건에 맞는 단지를 우선 노출한다.
3. 사용자는 단지별로 일반공급/특별공급 가능 여부를 확인한다.

### 시나리오 3: 일정 관리
1. 사용자는 관심 단지를 저장한다.
2. 서비스는 모집공고일, 특별공급일, 1순위 접수일, 발표일, 계약일을 정리해 보여준다.
3. 사용자는 알림을 설정해 일정을 놓치지 않는다.

### 시나리오 4: 당첨 가능성 판단
1. 사용자는 가점 계산기에서 무주택 기간, 부양가족 수, 통장 가입 기간을 입력한다.
2. 서비스는 총 가점을 계산하고, 과거 유사 단지의 당첨 가점 범위를 함께 보여준다.
3. 사용자는 관심 단지의 경쟁률과 내 가점을 비교해 지원 전략을 세운다.
4. 가점이 낮은 사용자에게는 추첨제 비율이 높은 단지나 공공분양을 안내한다.

### 시나리오 5: 자금 계획 수립
1. 사용자는 관심 단지 상세에서 특정 타입을 선택한다.
2. 서비스는 계약금, 중도금, 잔금의 금액과 예상 납입 시기를 보여준다.
3. 사용자는 자기 자금을 입력해 중도금 대출 필요액과 예상 이자를 확인한다.
4. 여러 단지의 자금 계획을 비교해 실현 가능성을 판단한다.

### 시나리오 6: 지도 기반 탐색
1. 사용자는 관심 지역의 지도를 확인한다.
2. 지도 위 마커로 분양 단지의 위치와 현재 단계를 파악한다.
3. 마커를 클릭해 간략 정보를 보고 관심 단지를 저장하거나 상세로 이동한다.

## 8. 핵심 기능 요구사항

### 8.1 분양 전용 리스트
목적:
- 사용자가 매매/전월세 정보 없이 분양 단지만 탐색할 수 있게 한다.

요구사항:
- 분양 관련 단지만 노출한다.
- 리스트 기본 정렬은 일정 임박 순 또는 현재 접수 가능 순으로 제공한다.
- 카드에는 단지명, 위치, 분양 단계, 공급 유형, 대표 분양가, 접수 일정이 포함된다.

### 8.2 분양 단계별 필터
목적:
- 사용자가 현재 행동 가능한 단지를 빠르게 찾게 한다.

필터 항목:
- 분양 예정
- 모집공고 공개
- 특별공급 접수
- 1순위 접수
- 2순위 접수
- 당첨자 발표
- 계약 진행
- 입주 예정

요구사항:
- 사용자는 한 개 또는 여러 단계 필터를 동시에 선택할 수 있다.
- 각 단지는 현재 기준으로 하나의 대표 단계를 가진다.
- 일정 경과에 따라 단계가 자동 갱신된다.

### 8.3 분양 맞춤 필터
목적:
- 청약 문맥에 맞는 검색 경험을 제공한다.

필터 항목:
- 지역
- 분양가 범위
- 전용면적
- 공급 유형(민간분양, 공공분양)
- 일반공급/특별공급
- 특별공급 유형(신혼부부, 생애최초, 다자녀, 노부모부양 등)
- 입주 예정 시기
- 규제지역 여부
- 세대수 규모
- 건설사/시행사

### 8.4 단지 상세 페이지
목적:
- 사용자가 청약 의사결정에 필요한 핵심 정보를 한 페이지에서 확인하게 한다.

포함 정보:
- 단지명, 위치, 지도
- 분양 단계
- 모집공고 요약
- 공급 세대수, 타입별 면적
- 분양가 및 계약금/중도금/잔금 구조
- 공급 유형 및 특별공급 정보
- 접수 일정, 발표일, 계약일, 입주 예정일
- 시행사/시공사
- 교통, 학군, 생활 인프라 요약
- 유의사항 및 공고문 원문 링크

### 8.5 청약 자격 간이 진단
목적:
- 사용자가 해당 단지를 볼 의미가 있는지 빠르게 판단하게 한다.

입력 항목:
- 무주택 여부
- 세대주 여부
- 거주 지역
- 청약통장 보유 여부 및 가입 기간
- 특별공급 대상 여부
- 소득/자산 조건 입력 여부(후속 확장)

출력:
- 지원 가능성이 있는 공급 유형
- 확인이 더 필요한 항목
- 단지별 주의사항

주의:
- 법률적 확정 판단이 아니라 사전 참고용으로 제공한다.

### 8.6 관심 단지 및 비교함
목적:
- 사용자가 여러 단지를 저장하고 비교할 수 있게 한다.

요구사항:
- 관심 단지 저장
- 최대 3~5개 단지 비교
- 비교 항목: 분양가, 입지, 단계, 공급 세대수, 일정, 입주 예정일

### 8.7 청약 가점 계산기
목적:
- 사용자가 자신의 청약 가점을 정확히 계산하고, 해당 가점으로 당첨 가능성이 있는 단지를 판단하게 한다.

입력 항목:
- 무주택 기간 (본인 및 배우자)
- 부양가족 수
- 청약통장 가입 기간

출력:
- 총 가점 (84점 만점 기준)
- 항목별 세부 점수
- 해당 가점으로 당첨 가능성이 있는 관심 단지 목록
- 과거 유사 지역/면적 당첨 가점 범위 참고 정보

요구사항:
- 가점제/추첨제 비율이 다른 단지를 구분해 안내한다.
- 계산 결과를 저장해 단지 상세에서 바로 활용할 수 있게 한다.
- 가점이 낮은 사용자에게는 추첨제 비율이 높은 단지를 안내한다.

### 8.8 경쟁률 정보
목적:
- 과거 분양 경쟁률 데이터를 제공해 사용자의 당첨 가능성 판단을 돕는다.

제공 정보:
- 해당 단지의 실시간/최종 경쟁률 (접수 중인 경우)
- 과거 유사 단지의 평균 경쟁률 (동일 지역, 유사 분양가, 유사 면적)
- 지역별 평균 경쟁률 추이
- 공급 유형별(일반/특별) 경쟁률 분리 표시
- 타입별(면적별) 경쟁률 분리 표시

요구사항:
- 경쟁률 데이터는 청약홈 공식 데이터를 기반으로 한다.
- 접수 중인 단지는 실시간 경쟁률 업데이트를 제공한다.
- 과거 데이터 기반 경쟁률 예상은 참고 정보임을 명시한다.

### 8.9 규제 정보 요약
목적:
- 단지별로 다른 규제 조건을 한눈에 파악하게 한다.

표시 항목:
- 분양가 상한제 적용 여부
- 전매제한 기간 (소유권 이전 등기일까지, 또는 N년)
- 실거주 의무 기간
- 재당첨 제한 기간
- 투기과열지구/조정대상지역 여부
- 중도금 대출 규제 여부

요구사항:
- 규제 조건은 단지 상세의 상단 영역에 아이콘/배지 형태로 요약 표시한다.
- 각 규제 항목 클릭 시 쉬운 해설을 제공한다.
- 규제 변경 시 알림을 보낼 수 있는 구조로 설계한다.

### 8.10 자금 계획 시뮬레이션
목적:
- 분양가 기준으로 계약금, 중도금, 잔금 납입 시기와 금액을 시뮬레이션하게 한다.

입력 항목:
- 관심 타입 선택
- 자기 자금 입력 (선택)

출력:
- 계약금/중도금/잔금 금액 및 납입 예정 시기
- 중도금 대출 예상 필요액
- 중도금 이자 예상 (기본 금리 기준)
- 총 필요 자금 요약

주의:
- 대출 금리는 참고용 평균 기준이며 실제와 다를 수 있음을 명시한다.
- MVP에서는 단순 계산만 제공하고, 정밀 대출 한도 계산은 후속 확장으로 둔다.

### 8.11 지도 기반 분양 탐색
목적:
- 사용자가 관심 지역의 분양 단지를 지도 위에서 직관적으로 탐색하게 한다.

요구사항:
- 지도 위에 분양 단지를 마커로 표시한다.
- 마커 색상은 분양 단계에 따라 구분한다.
- 마커 클릭 시 간략 정보 카드(단지명, 단계, 분양가, 일정)를 보여준다.
- 지도 영역 이동/줌에 따라 리스트가 연동된다.
- 리스트 뷰/지도 뷰 전환이 가능하다.
- 분양 찾기의 모든 필터가 지도 뷰에도 동일하게 적용된다.

### 8.12 온보딩

목적:
- 신규 사용자가 서비스 핵심 가치를 빠르게 이해하고, 개인화에 필요한 조건을 자연스럽게 입력하게 한다.

플로우:
1. 서비스 소개 (분양 전용 서비스임을 인지, 2~3 스크린)
2. 관심 지역 선택
3. 예산 범위 입력
4. 무주택/세대주/청약통장 여부 간편 입력
5. 특별공급 대상 여부 선택
6. 완료 → 맞춤 추천 단지 홈으로 이동

요구사항:
- 각 단계는 건너뛰기가 가능해야 한다.
- 비로그인 상태에서도 입력 가능하고 로컬에 저장한다.
- 이후 로그인 시 서버에 동기화한다.

### 8.13 일정 캘린더 및 알림
목적:
- 사용자가 청약 일정을 놓치지 않게 한다.

요구사항:
- 관심 단지의 주요 일정 캘린더 표시
- D-day 표시
- 푸시/이메일/문자 알림 확장 가능 구조
- 모집공고, 접수 시작, 발표일, 계약일 알림 제공

## 9. 비기능 요구사항

### 성능
- 리스트 및 필터 응답은 빠르게 동작해야 한다.
- 모바일에서도 핵심 정보 접근이 쉬워야 한다.

### 신뢰성
- 일정, 단계, 공급 정보는 최신 상태를 유지해야 한다.
- 데이터 업데이트 시점이 사용자에게 표시되어야 한다.

### 확장성
- 향후 오피스텔, 지식산업센터, 공공분양 등 자산군 확장이 가능해야 한다.
- 청약 자격 계산 로직은 규정 변경에 대응 가능해야 한다.

### 법적/정책적 주의
- 자격 진단 결과는 참고 정보임을 명확히 표기한다.
- 금융/법률 확정 자문으로 오해될 표현을 피한다.

- 분양 맞춤 필터

- 경쟁률 정보 (과거 데이터 기반)

## 13. 출시 후 확장 방향

### 1단계
- 자금 계획 시뮬레이션 (계약금~잔금 납입 계획)
- 실시간 경쟁률 업데이트 (접수 기간 중)
- 지역별 인기 분양 랭킹
- 조건 기반 추천 단지 고도화
- 공고문 AI 요약

### 2단계
- 주변 시세와 분양가 비교 분석
- 캘린더 외부 연동 (Google Calendar, Apple Calendar)
- 분양권 전매 정보 제공
- 청약 전략 리포트 (가점제/추첨제 최적 전략)
- 커뮤니티 (단지별 관심자 Q&A)

### 3단계
- AI 기반 당첨 확률 예측
- 개인 맞춤형 우선순위 추천
- 분양 후 계약/입주 준비 콘텐츠
- 대출 한도 정밀 계산 (금융기관 연동)
- B2B 분양 광고 플랫폼

## 14. 제품 원칙

- 분양 외 정보는 과감히 제외한다.
- 사용자가 지금 행동할 수 있는 정보를 우선 노출한다.
- 복잡한 청약 정보를 쉬운 언어로 번역한다.
- 탐색보다 실행에 도움이 되는 구조를 우선한다.
- 최신성과 신뢰성을 가장 중요한 품질 기준으로 둔다.
