# AGENTS.md

이 문서는 이 저장소에서 작업하는 AI 에이전트와 기여자를 위한 공통 규칙이다.

## 1. 작업 전 문서 컨텍스트 확인

작업을 시작하기 전에 현재 작업과 관련된 기존 문서를 확인한다. 중복 로직, 기능 상충, 용어 불일치를 방지하기 위한 필수 절차다.

우선 확인 대상 문서:
- `PRD.md`
- `TECH_ARCHITECTURE.md`
- `DATA_MODEL.md`
- `SCREEN_SPEC.md`
- `WIREFRAMES.md`
- `CLAUDE.md`

모든 문서를 항상 전부 읽을 필요는 없지만, 현재 작업과 직접 연결되는 문서는 반드시 확인한다.

## 2. 작업 계획 기록

작업을 진행할 때에는 먼저 `ai/plan/` 아래에 작업 계획 문서를 작성한다.

권장 규칙:
- 파일명은 날짜와 작업명을 포함해 명확하게 작성한다.
- 계획 문서에는 최소한 아래 내용을 포함한다.
  - 작업 목표
  - 참고한 문서
  - 영향 범위
  - 구현 단계
  - 리스크 또는 가정

예시:
- `ai/plan/2026-03-09-offerings-filter-update.md`

## 3. 구현 원칙

코드를 작성할 때는 유지보수에 용이한 구조를 우선한다.

필수 원칙:
- 책임이 섞이지 않도록 기능을 작은 단위로 분리한다.
- 이름은 도메인 문서의 용어와 일치시킨다.
- 중복 로직은 복사하지 말고 공통화 가능한 구조를 우선 검토한다.
- 임시 처리나 하드코딩이 필요하면 이유와 제거 조건을 문서화한다.
- 현재 요구사항을 넘는 과도한 추상화는 피하되, 이후 확장 가능성은 고려한다.
- 변경 시 기존 기능과 충돌하지 않는지 관련 문서와 구현을 함께 확인한다.

## 4. 문서 기반 검증

신규 기능, 정책, 데이터 구조, 화면 흐름을 다룰 때는 기존 문서 컨텍스트와 대조하여 아래 항목을 확인한다.

- 이미 정의된 기능인지
- 다른 화면/모듈과 충돌하지 않는지
- 데이터 모델과 용어가 일치하는지
- 기술 아키텍처 방향과 어긋나지 않는지

문서 간 충돌이나 미정 사항이 있으면 임의로 확정하지 말고 작업 계획 또는 작업 로그에 가정과 확인 필요 사항을 남긴다.

## 5. 작업 완료 로그 기록

작업이 완료되면 `ai/work/` 아래에 작업 로그 문서를 작성한다.

작업 로그에는 최소한 아래 내용을 포함한다.
- 실제 변경 사항
- 참고한 문서
- 주요 결정 사항
- 검증 또는 확인 내용
- 후속으로 확인할 이슈가 있다면 그 내용

예시:
- `ai/work/2026-03-09-offerings-filter-update.md`

## 6. 기본 기대 사항

이 저장소의 작업은 단순 구현보다 문서와의 정합성이 중요하다. 모든 변경은 기존 문맥을 존중하면서, 이후 작업자가 빠르게 이해하고 이어서 유지보수할 수 있는 상태를 목표로 한다.
