# [07] 온보딩 서비스 소개 슬라이드 (1단계) 추가

## 작업 목표
현재 온보딩이 바로 입력 폼으로 시작하는 것을 기획서 기준에 맞게 수정한다.
첫 단계에 서비스 소개 스와이프 카드 2~3장을 추가한다.

## 참고 문서
- `docs/SCREEN_SPEC.md` — 5.0 온보딩, 단계 구성
- `docs/WIREFRAMES.md` — 온보딩 와이어프레임

## 영향 범위
- `apps/web/src/pages/OnboardingPage.tsx` — 슬라이드 단계 추가

## 구현 단계

### 1. 슬라이드 데이터 정의
기획서 기준 3장:
1. "분양과 청약에 필요한 정보만 모았어요" — 분양 전용 서비스 소개
2. "내 조건에 맞는 단지를 추천받으세요" — 개인화 추천 소개
3. "일정 알림으로 청약을 놓치지 마세요" — 알림 기능 소개

각 슬라이드: 제목, 설명 텍스트, 일러스트/아이콘 영역

### 2. OnboardingPage 수정
- 기존 step 0~2 (입력 폼)를 step 1~3으로 밀어냄
- 새 step 0: 서비스 소개 슬라이드 (내부에서 카드 index 0~2 스와이프)
- 슬라이드에서는 "시작하기" 버튼으로 step 1(입력 폼)으로 이동
- 슬라이드에서도 "건너뛰기" 가능

### 3. 슬라이드 인터랙션
- 터치/마우스 스와이프 or 버튼으로 카드 전환
- 하단 dot indicator로 현재 카드 위치 표시
- 마지막 슬라이드에서 "시작하기" 버튼 강조

### 4. 진행 상태 표시
- 전체 TOTAL_STEPS에 슬라이드 단계 포함 여부 결정
  - 안: 슬라이드는 별도 인트로로 처리, 프로그레스 바는 입력 단계부터 카운트

## 리스크 / 가정
- 슬라이드 애니메이션은 CSS transition으로 간단하게 구현 (라이브러리 추가 최소화)
- 이미 온보딩을 완료한 사용자는 슬라이드를 다시 보지 않음 (completeOnboarding 플래그 활용)

## 완료 조건
- 첫 진입 시 서비스 소개 슬라이드 3장이 표시됨
- 스와이프 또는 버튼으로 카드 전환 가능
- "시작하기" 또는 "건너뛰기"로 입력 단계로 이동
- 기존 3단계 입력 플로우는 동작 그대로 유지
