# 작업 로그: AGENTS 규칙 추가

## 작업 내용
- 루트에 `AGENTS.md`를 신규 작성했다.
- 작업 계획 작성 규칙을 `ai/plan/` 기준으로 문서화했다.
- 작업 완료 로그 작성 규칙을 `ai/work/` 기준으로 문서화했다.
- 유지보수 용이성, 문서 컨텍스트 확인, 중복 로직 및 기능 상충 방지 원칙을 반영했다.

## 참고 문서
- `CLAUDE.md`
- `PRD.md`
- `TECH_ARCHITECTURE.md`

## 주요 결정
- 파일명 규칙은 날짜와 작업명을 포함하는 형태로 제안했다.
- 문서 확인 대상을 명시해 작업 전에 무엇을 참고해야 하는지 바로 알 수 있게 했다.
- 구현 원칙과 문서 기반 검증 절차를 분리해 실무 적용이 쉽도록 정리했다.

## 확인 내용
- `ai/plan/`과 `ai/work/` 디렉터리가 이미 존재하는 것을 확인했다.
- 저장소는 현재 기획 문서 중심 구조이며, 코드 작성 전 문서 컨텍스트 확인이 핵심이라는 점을 반영했다.

## 후속 메모
- 이후 실제 코드가 추가되면 테스트/검증 규칙도 `AGENTS.md`에 확장할 수 있다.
